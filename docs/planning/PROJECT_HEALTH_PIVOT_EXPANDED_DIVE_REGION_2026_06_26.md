# Project Health Pivot - Expanded Dive Region - 2026-06-26

Status: active direction reset after project evaluation.

## Why This Pivot Exists

Recent work made the first vertical slice safer and more readable, but the active queue drifted too far toward polish, reviews, and repeated evidence refreshes. Playtesting now needs more player-visible progress: wider exploration, side routes, cave/wreck pockets, and a larger expedition shape.

The current slice is good enough to move forward as a foundation. Do not keep optimizing the same early pressure/predator/reward cluster unless it blocks the expanded region milestone.

## New Priority

Build the first expanded dive-region prototype:

- wider side-view navigation,
- camera/world bounds that support horizontal exploration,
- one side-route branch,
- one cave or wreck-pocket entrance,
- one stable landmark for orientation,
- one variable daily opportunity in the side route,
- one upgrade-locked promise for returning later,
- safe-return readability from the new branch,
- cheap test tiers so validation does not waste time or tokens.

## Workflow Reset

Default verification should be cheap:

- quick tier for code changes,
- docs tier for planning/docs changes,
- visual tier only for UI/art/camera/route visual changes,
- full visual/export tier only before major milestone closeout or when export/screenshot tooling changes.

Backlog refreshes and MCP refreshes should not be used as frequent filler. Use them only when the queue is genuinely low or context resources changed materially.

## Deferred From The Previous Queue

The art-kit tuning batch #424-#443 is intentionally deferred. Those issues can be revisited after the first expanded region makes the playable map feel meaningfully larger.

## Non-Goals

- No final production art lock.
- No broad procedural ocean.
- No minimap, field guide, quest checklist, exact locator, combat, capture, harvesting, or base-management layer.
- No broad scene rewrite before the first expanded-region prototype proves the shape.
