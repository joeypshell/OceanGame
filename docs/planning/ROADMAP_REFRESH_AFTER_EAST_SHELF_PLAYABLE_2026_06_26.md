# Roadmap Refresh After East Shelf Playable - 2026-06-26

Status: current roadmap refresh.

## Purpose

Record that `East Shelf Spur` is no longer only a spatial prototype. It now has enough interaction, payoff, result memory, timing/navigation behavior, and lower-connector setup to count as the first minimal playable side-route sequence.

This refresh keeps upcoming work pointed at player-visible map progress instead of returning to broad first-screen polish.

## What Changed

`East Shelf Spur` now includes:

- a right-side route scaffold with camera/world bounds,
- `East Shelf Arch` landmark and up-left return orientation,
- seeded `Shelf Glimmer` rare-signal opportunity,
- `PocketEntrance` prompt and one run-scoped research payoff,
- extraction result memory for the pocket ping,
- a non-combat current-surge timing cue,
- `Sealed Shelf Hatch` as a locked future route promise,
- `Shelf Drop Connector` as a short lower continuation,
- `Drop Arch` landmark and return-current cue,
- `Drop Echo` as one lower-connector research opportunity,
- oxygen-margin, reset/bounds, no-minimap, and Playwright route-stage guardrails.

## Roadmap Meaning

The active milestone is now past "can the map widen?" and into "can wider routes create repeatable dive decisions?"

The next work should prioritize:

1. a clearer second reason to choose East Shelf or Shelf Drop on a daily run,
2. a first sealed-hatch access step that remains narrow,
3. one more connected pocket or branch below/around Shelf Drop,
4. better route readability only where new route content makes the screen unclear,
5. cheap automated state and screenshot evidence for each route addition.

## What Not To Do Next

Do not turn this refresh into:

- another broad HUD polish queue,
- final art lock,
- full procedural ocean,
- minimap,
- quest checklist,
- field-guide objective system,
- exact locator,
- combat/capture/harvesting system,
- broad upgrade tier expansion,
- base-management layer.

## Success Criteria For The Next Batch

The next issue batch should make the playable ocean feel bigger in the actual game. At least half of the next backlog should be implementation issues that add or test concrete route behavior, route readability, or progression promises. Review-only issues should exist only when they directly unblock those implementation issues.
