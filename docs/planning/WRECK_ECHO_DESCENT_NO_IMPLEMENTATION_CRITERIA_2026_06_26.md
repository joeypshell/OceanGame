# Wreck Echo Descent No-Implementation Criteria - 2026-06-26

Status: planning guardrail for issue #304.

## Purpose

Define what Wreck Echo Descent planning may do right now, and what it must not do.

This protects the current playable slice from accidentally gaining a new route, locator system, quest chain, or broad region before the existing Wreck Shelf, pressure gate, cache, predator lane, HUD, and return loop are fully readable.

Allowed planning anchors:

- `Echo Lens I` as broad wreck-echo curiosity, not exact navigation.
- `Pressure Seal I` as proof that pressure access can open remembered places.
- Wreck Shelf memory as the existing blocked-promise region.
- Oxygen margin as preparation pressure.
- Cargo space as a possible future route tradeoff.
- Daily/seeded expedition framing as a reason a future run may feel tempting.
- No-locator rules from `MYSTERY_SPINE_V1.md` and `SCANNER_PROGRESSION_V2.md`.

## Planning Must Not Produce

- New `Main.tscn` route nodes.
- New scan targets.
- New resource pickups.
- New collision, pressure boundary, or denial zone.
- New persistent marker, minimap node, route graph, field-guide entry, quest log, or objective checklist.
- New exact direction/distance tracker.
- New predator behavior.
- New upgrade or resource economy.
- New biome/region implementation.
- Scene-wired pressure-dark threshold art.

## Required Before Any Future Implementation Issue

Before Wreck Echo Descent can become implementation work, a future issue must prove:

- current vertical-slice acceptance has no blocker-level readability failures,
- lower-route pressure gate/cache/predator overlap is pass or pass-with-low-risk watchlist,
- active HUD low/critical oxygen remains louder than reward, mystery, and background art,
- Echo Lens copy remains broad/local/category-based and avoids exact navigation,
- the route has one scoped data/scene ownership plan,
- the route has one selected reward type, not a multi-objective chain,
- pressure access remains a real requirement and is not bypassed by scanner curiosity,
- oxygen/cargo preparation is understandable without a checklist,
- safe return remains readable before, during, and after the route attempt.

## Future Implementation Acceptance Must Include

If a later issue explicitly authorizes implementation, its acceptance criteria must include:

- no minimap, route graph, exact locator, field guide, quest log, or objective checklist,
- no broad procedural biome generation,
- no new combat, predator harvesting, capture, or loot loop,
- no pressure bypass from `Echo Lens I`,
- no persistent marker after the scan/result moment,
- one route pocket, one route question, one reward choice,
- result copy that preserves memory in one compact line,
- screenshots or normal-scale evidence showing current HUD, return, pressure, cache, and predator cues still readable.
