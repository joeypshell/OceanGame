- 8 named regions, including open surface water, surface seafloor, starter cave, Shell Reef, Thermal Vent, Blue Chimney, Pressure Wreck, and future deep exit.
- 13 solid terrain polygons copied from the accepted surface-floor geometry.
- Runtime generation metadata for each terrain polygon:
  - `Polygon2D` terrain fill name;
  - `StaticBody2D` blocker name;

  - rim/trim container name;
  - terrain-kit source.

- Merged validation rules from the old blockout rescue map and the accepted source-floor geometry.

## Semantic Trim Editing Rule

Add terrain sprites only through named source-map trim segments. A valid trim segment must identify:

- the terrain polygon id it belongs to;
- the terrain role it represents;
- the exact start/end points in map coordinates, on that terrain polygon's source edge;
- offset, repeat spacing, scale, and alpha;
- whether it is a visual-only trim.

Do not place terrain sprites by eyeballing screen coordinates in `Main.tscn`. That breaks repeatability and makes collision/visual drift likely. The docs validation tier now runs `tools/validate-area01-runtime-source-map.mjs` to catch missing trim segments, unsupported trim roles, duplicate ids, and trim endpoints that drift away from authoritative terrain polygon edges.

## Source-Of-Truth Rule

For runtime v2, the intended invariant is:

```text
source polygon == visible terrain polygon == collision polygon source == rim/trim placement source
```

Sprites do not define collision. Preview PNGs are QA artifacts only.
