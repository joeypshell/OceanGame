# Shell Reef Route Review

Status: current checklist for validating the first small biome pocket.

Use this checklist after Shell Reef changes when judging whether the pocket changes expedition decisions. This is a solo or facilitator-friendly review tool. It does not revive the skipped blind-validation gate.

## Purpose

The Shell Reef should do more than decorate the midwater band. A pass should answer whether the reef creates a meaningful choice:

- bank midwater cargo safely,
- scan and use the reef clue,
- leave the reef with partial cargo and push deeper toward pressure, predator, or glow rewards,
- return early because oxygen, cargo, or route risk is already tight.

## Setup

1. Launch the project normally.
2. Keep F3 telemetry available for seed, cluster pattern, cargo, scans, contacts, oxygen result, and failure cause.
3. Run five consecutive expeditions from a clean-enough state for the goal of the review. Full save reset is optional, but record notable upgrades already owned.
4. Do not use this as a pass/fail gate for outside-player validation. Convert concrete observations into follow-up issues.

## Pass Signals

Treat the reef as working when most of these are true across five seeds:

- At least one expedition makes midwater banking through the reef feel sensible.
- At least one expedition tempts the player to pass the reef and push deeper.
- The Shell Reef Shelf scan clue changes or confirms a route decision.
- Shell placement stays midwater and does not trivialize upgrade costs.
- The reef reads as a landmark without hiding the surface/base return direction.
- Predator contact remains avoidable and is not forced by the reef layout.
- The route decision changes between cautious and deep-reward cluster patterns.

## Fail Signals

Create follow-up issues if any of these show up:

- The reef is noticed only as decoration and does not affect route choice.
- Shell placement makes upgrade costs too easy or repetitive.
- The reef visually blocks the return path or makes the base direction confusing.
- The reef pushes the player into unavoidable `Gulper Eel` contact.
- The scan clue is too vague to change behavior.
- The player always ignores the reef because deeper rewards dominate.
- The player always stops at the reef because deeper rewards are not visible or feel too punishing.

## Follow-Up Issue Shapes

Use concrete observations, not broad polish requests:

- `Improve Shell Reef return readability after seed <seed>`
- `Tune Shell Reef shell candidate count after five-seed review`
- `Strengthen Shell Reef scan clue route guidance`
- `Adjust deep-reward lure relative to Shell Reef route`
- `Move predator warning away from Shell Reef return lane`
